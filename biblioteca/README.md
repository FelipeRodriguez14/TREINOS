ESSE É UM PROJETO SIMPLES, QUE REPRESENTA UM PEQUENO SISTEMA DE UMA BIBLIOTECA.

PORÉM, MESMO SENDO SIMPLES, O PROJETO ABORDA TODOS OS CONCEITOS INICIAS DE LÓGICA DE PROGRAMÇÃO E 
PROGRAMAÇÃO ORIENTADA A OBJETOS.

TÓPICOS ABORDADOS:

- ESTRUTURAS CONDICIONAIS
- ESTRUTURAS DE REPETIÇÃO
- ENCAPSULAMENTO
- CRIAÇÃO E TRATAMENTO PERSONALIZADO DE EXCEÇÕES
- ESTRUTURA BÁSICA DE UM PROJETO (ENTIDADES, SERVICES, EXCEPTIONS e ETC)
- CONCEITO DE CRUD (CREATE, READ, UPDATE, DELETE)

ALGUNS MÉTODOS PODEM ESTAR COM AS LÓGICAS UM POUCO EXAGERADAS PARA FUNCIONAR, MAS FIQUE A 
VONTADE PARA ALTERAR O CÓDIGO E ESTUDAR O PROJETO COMO PREFERIR.

*** NOTAS ***

* TALVEZ EU TENHA COLOCADO MUITA COISA SEM NECESSIDADE NOS MENUS.
* AS CRIAÇÕES DE CADA CLASSE MENU PODE TER FICADO UM POUCO BAGUNÇADA, MAS AQUI ESTÁ A DEFINIÇÃO:

MainAdmin: ESSE É O MENU PRINCIPAL DE QUANDO O USUÁRIO LOGA COMO ADMINISTRADOR. 
Mainuser: ESSE É O MENU PRINCIPAL CASO O USUÁRIO NÃO SEJA ADMINISTRADOR.
MainIniciar: ESSE É O MENU INICIAL, ONDE O USUÁRIO ESCOLHE COMO IRÁ FAZER LOGIN NO SISTEMA.

MenuServices: A CRIAÇÃO DESSA CLASSE TEM O INTUITO DE ORGANIZAR O MOMENTO EM QUE CADA "MainMenu" ACIMA 
IRÁ APARECER. PORÉM, OS MÉTODOS DESSA CLASSE FICARAM COM OS NOMES PARECIDOS COM AS CLASSES ACIMA, O QUE 
PODE GERAR UMA CONFUSÃO. DESCOBRI ISSO SOMENTE DA METADE DO PROJETO PRA FRENTE, ENTÃO PREFERI DOCUMENTAR 
AQUI DO QUE ORGANIZAR TUDO NOVAMENTE.
